Playing against itself

Training the neural network from self-play data

Each training iteration does this:

    ↓

    ↓
Train neural network on stored data
    ↓
Repeat


Now let’s break every part down.

♟️ 1. State Representation (encode.py)
What is a “state”?

A chess board position must be turned into numbers so a neural network can process it.

You convert the board into:

(18, 8, 8) tensor


Planes:

Plane	Meaning
0–5	White P, N, B, R, Q, K
6–11	Black P, N, B, R, Q, K
12	Side to move
13–16	Castling rights
17	En-passant file

So each square becomes a stack of binary features.

Example:
If white knight on g1 → plane[1, row, col] = 1.

This is exactly how AlphaZero represents boards.

🎯 2. Action Space

Your action space size:

64 from-squares × 64 to-squares × 5 promotion types
= 20480 actions


Every possible move maps to an index:

index = ((from * 64) + to) * 5 + promo


This makes the neural network output a fixed-size vector.

Even if only ~30 moves are legal, the output size is always 20480.

Illegal moves are masked.

🧠 3. Neural Network (net.py)

The model predicts two things:

Input: board tensor
Output:
    1) policy logits (20480)
    2) value scalar in [-1, 1]

Architecture

Stem:

Conv → BN → ReLU


Residual blocks:

Conv → BN → ReLU
Conv → BN
+ skip connection
ReLU


This lets the network go deeper without vanishing gradients.

Then two heads:

Policy Head

Predicts probability of each action.

Used to guide MCTS.

Value Head

Predicts expected game outcome from current position.

Used during tree backup.

🌳 4. MCTS (mcts.py)

This is the core intelligence.

Each move decision uses Monte Carlo Tree Search.

Let’s break one move into pieces.

Root Node

When it’s time to choose a move:

root = Node(current_board)
run M simulations
build visit distribution π
sample action

What is a Node?

Each Node stores:

P[a]  prior from neural net
N[a]  visit count
W[a]  total value
Q[a] = W[a] / N[a]
children[a]  → child node


It also stores:

legal actions

board state

sum_N

🔄 What happens in one simulation?

Each simulation does:

1️⃣ Selection

Walk down the tree using:

Q(a) + c_puct * P(a) * sqrt(total_visits) / (1 + N(a))


This balances:

Exploitation → Q

Exploration → P

2️⃣ Expansion

When reaching a node not expanded yet:

Call neural network

Get policy + value

Store priors P[a]

Store legal actions

3️⃣ Evaluation

Neural network outputs:

value v ∈ [-1, 1]


Meaning:
+1 → current player winning
-1 → losing

4️⃣ Backup

Walk back up the path:

parent.W[a] += v
parent.N[a] += 1
v = -v   # flip perspective each ply


Why flip?
Because players alternate.

📊 After all simulations

We compute:

π[a] = N[a] / sum_N


This is the improved policy.

Not the raw neural net output —
but a search-refined distribution.

This π is what we train on.

🔁 Tree Reuse

After selecting a move:

root = root.children[action]


So we reuse subtree knowledge.

This dramatically speeds up search.

🎮 5. Self-Play (selfplay.py)

For each move:

Run MCTS

Store:

state

π (visit distribution)

whose turn it was

Sample move

Play move

Reuse tree

At game end:

Get final result z

Assign value target:

If white to move at state → v = z

If black to move → v = -z

So each stored example is:

(state, π, v)

📦 6. Replay Buffer

You store training samples in:

ReplayBuffer


And persist them to disk as:

replay/shard_xxx.npz


Each shard contains:

states

π distributions

values

This allows stopping/resuming training.

🏋️ 7. Training Step

Each batch:

logits, v_pred = net(states)


Loss:

Policy loss

Soft cross-entropy:

-(π * log_softmax(logits))


You’re teaching the net:
“Match MCTS behavior.”

Value loss

MSE:

(v_pred - v_target)^2


Total:

policy_loss + value_loss


Then backprop.

📈 8. Evaluation vs Random

Every few iterations:

Play games vs random player

Measure win/draw/loss

Log to TensorBoard

This tells you if learning is real.

🚀 What Actually Learns?

At start:

Policy = random

Value = random

So MCTS behaves randomly.

After training:

Policy predicts stronger moves

Value predicts better outcomes

MCTS becomes stronger

Stronger self-play data

Network improves further

This is a feedback loop:

Network → Search → Better Data → Better Network

⚡ Why It’s Slow

Because:

For each move:
    15–25 simulations
        each simulation:
            tree walk
            NN inference


Multiply by:

~100 plies per game

multiple games per iteration

That’s heavy CPU work.

🧠 The Most Important Concept

The neural network is NOT directly picking moves.

It is:

guiding search

learning from search

gradually internalizing search knowledge

Eventually, the network becomes strong enough that search becomes shallow.
